LESS Scope & Extend

LESS Scope

The concept of scope in LESS, is similar to the concept of scope in programming languages broadly.

When you declare variables and use them in a nested structure, the complier will look for the variable in the local scope first.

If it isn't found locally, it will then look for a value in the elements parent scope, and continue up the scope chain until it either find's a definition for the variable, or doesn't.

LESS Extend

You can use the extend concept in LESS to share properties from, one class or selector to the next.

It is essentially the same concept wise, as a mixin, but has slighty different syntax.

You use it with the syntax &:extend(selector)
