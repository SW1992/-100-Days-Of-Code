LESS Nesting & Operations

LESS Nesting

Nesting in LESS, is a way of reducing & simplyfing the syntax of CSS Combinators when accessing child elements, down to a level that's more representative of their hierarchical form.

For instance, to access the an a, inside a li, inside a ul, and style it's when it's visited & when it's hovered over, in normal CSS you'd have to write something like

ul li a:visited {} ul li a:hover {}

With high levels of nesting & different pseduo selectors, this could all become quite confusing to look at.

with LESS nesting you could access it like

ul { li { a { &:visited { } } &hover: { } } } }

LESS Operations

LESS Operations, are a way of performing mathematical operations with LESS Variables.

They have many uses, you could use it to set the padding-left of a group of elements relative to each other for example.

LESS Operations are generally performed within parentheses.
